# Pure Episodic Memory Navigation

このディレクトリは、訪問回数を使わない純粋なエピソード記憶ベースのナビゲーション実験を含みます。

## 実験の動機

「エピソード記憶から、訪問回数抜いてもマルチホップ評価があれば有効に迷路解けるんじゃない？」という仮説を検証。

## 主要ファイル

### 実装
- `pure_episodic_navigator.py` - メインの実装（15×15で成功）
- `optimized_episodic_navigator.py` - 大規模迷路向け最適化版
- `pure_episodic_v2.py` - 改良版実装

### 実験・分析
- `test_small_maze.py` - 小規模迷路でのテスト
- `test_larger_maze.py` - 大規模迷路でのスケーリングテスト
- `debug_episodic.py` - デバッグ用分析ツール
- `EXPERIMENT_DESIGN.md` - 詳細な実験設計
- `RESULTS.md` - 実験結果のまとめ

### 可視化
- `test_small_maze.png` - 15×15迷路の解法可視化
- `visualizations/` - その他の可視化結果

## 主要な成果

1. **訪問回数なしでの学習を実証**
   - 15×15迷路を59ステップで解決
   - マルチホップ評価が鍵

2. **ゴール信号の伝播**
   - メッセージパッシングによる情報共有
   - 3-hop評価が最も効果的（75%の決定で使用）

3. **理論的意義**
   - geDIGが記憶選択関数として機能
   - 生物学的に妥当なメカニズム

## 実行方法

```bash
# 小規模テスト
python test_small_maze.py

# 大規模テスト
python test_larger_maze.py
```

## 技術的詳細

### エンベディング設計（6次元）
1. 位置X（正規化）
2. 位置Y（正規化）
3. 行動（-1.0〜1.0）
4. 結果（成功: 1.0, 壁: -1.0, 訪問済: 0.0）
5. 局所トポロジー（壁の数から計算）
6. ゴール信号（0 or 10）

### マルチホップ評価
- 1-hop: 直接の経験
- 2-hop: 近傍の経験を考慮
- 3-hop: より広範囲の文脈を考慮
