# 迷路実験の理論的前提と革新性
*InsightSpike-AI Maze Navigation Experiment*

## 🎯 なぜ迷路実験が重要なのか

### 1. 純粋エピソード記憶への挑戦

従来の迷路探索手法：
- **強化学習（RL）**: 報酬信号が必要
- **A*アルゴリズム**: 事前の地図情報が必要
- **SLAM**: センサー情報の統合が必要

**我々のアプローチ：**
```
外部報酬なし + 地図なし + 純粋な記憶のみ = 真の自律探索
```

これは人間の空間認知により近い、**生物学的に妥当**なアプローチです。

### 2. geDIG理論の完璧な検証環境

迷路探索は以下の理由でgeDIG理論の検証に最適：

| 特性 | 迷路での意味 | geDIG検証への寄与 |
|------|------------|------------------|
| **構造的変化が明確** | 分岐・行き止まり・ループ | ΔGEDの変化が観測しやすい |
| **情報の局所性** | 各位置での観測が限定的 | ΔIGの計算が明確 |
| **最適解の存在** | 最短経路が定義可能 | 理論的な性能上限が明確 |
| **再現性** | 決定論的環境 | 科学的検証が可能 |

## 🧠 エピソード記憶の革新的表現

### 8次元ベクトル設計の理論的根拠

```python
vector = [
    x/width,        # 空間位置（正規化）
    y/height,       # 空間位置（正規化）
    dx,             # 運動方向
    dy,             # 運動方向
    wall_flag,      # 環境制約
    log_visits,     # 時間的記憶
    reserved,       # 将来の感覚情報
    reserved        # 将来のコンテキスト
]
```

この表現は：
1. **位置情報**（次元0-1）: 空間的アイデンティティ
2. **方向情報**（次元2-3）: 運動の意図
3. **環境情報**（次元4）: 物理的制約
4. **履歴情報**（次元5）: 時間的経験

これらは脳の**格子細胞**、**場所細胞**、**頭方向細胞**に対応します。

## 🔬 geDIG値の空間探索での意味

### 正のgeDIG値（探索フェーズ）
```
geDIG > 0 → 新しい構造の発見
```
- 未知の分岐点を発見
- 新しい経路の開拓
- 探索空間の拡大

### 負のgeDIG値（統合フェーズ）
```
geDIG < 0 → 既存構造の最適化
```
- ショートカットの発見
- 冗長経路の削除
- 知識の圧縮

### ゼロ付近（活用フェーズ）
```
geDIG ≈ 0 → 安定した知識活用
```
- 既知経路の効率的利用
- 最適経路の選択

## 📊 既存手法との本質的な違い

### MFEC（Model-Free Episodic Control）との比較

| 項目 | MFEC | geDIG-Navigator |
|------|------|----------------|
| メモリ成長 | O(n) - 無限増大 | O(log n) - 自動圧縮 |
| 構造理解 | なし（個別記憶） | あり（グラフ統合） |
| 忘却機構 | 固定サイズLRU | 適応的統合 |
| 汎化能力 | 低（exact match） | 高（構造的類似性） |

### NEC（Neural Episodic Control）との比較

| 項目 | NEC | geDIG-Navigator |
|------|-----|----------------|
| ニューラルネット依存 | 必須（DNN） | 不要（純粋な数学） |
| 学習時間 | 長い（勾配降下） | 即座（構造更新） |
| 解釈可能性 | 低（ブラックボックス） | 高（グラフ可視化） |

|--------|-----------|

メモリ複雑度の理論的保証：


これは**無限に大きな迷路**でも実用的なメモリ使用量で動作することを意味します。
