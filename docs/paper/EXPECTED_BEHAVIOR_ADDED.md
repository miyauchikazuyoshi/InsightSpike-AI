## 記述した2つのケース

### ケース1: 移動可能エピソードが存在する場合
```
観測範囲内に移動可能なエピソード有り
  ↓
ソフトマックス確率で行動決定
  ↓
既知経路を辿る（グラフ変化小）
  ↓
geDIG値は安定
```

**意義**: 通常動作の説明

### ケース2: 移動可能エピソードが存在しない場合 ⭐重要
```
1. 類似エピソード選択
   └─ T字路・十字路で選択しなかった分岐
   └─ 空間距離が最も近いものを選択

2. エッジ結線 & 0-hop評価
   ├─ GED: 大きい（新規追加）
   ├─ IG: 増加（新情報）
   ├─ g₀: ゼロ付近（曖昧さ検出）
   └─ → NA発火！

3. Multi-hop評価
   ├─ 遠方ノード間の接続評価
   ├─ グラフの短絡を観測
   ├─ 平均最短路長↓（ΔSP < 0）
   ├─ GED実効値↓（効率化）
   └─ g_min: 大きな負値（洞察！）

4. BT発火 & バックトラック
   ├─ b(t) = min{g₀, g_min} ≤ θ_BT
   ├─ グラフ内経路をBFS探索
   ├─ 結線エピソードへのパス想起
   └─ バックトラック実行

5. 探索再開
   └─ 新経路から継続
```

## 追加した図解説明

### 動作フローの5段階
1. **類似エピソード選択**: T字路・十字路の選択しなかった分岐
2. **0-hop評価**: 曖昧さ検出（g₀ ≈ 0）→ NA発火
3. **Multi-hop評価**: 短絡検出（g_min << 0）→ 洞察
4. **BT発火**: 二値閾値評価 → バックトラック
5. **探索再開**: 効率的な経路探索

### エージェントが得るもの
- ✅ 局所的な行き止まりで「曖昧さ」を検知（NA）
- ✅ 遠方との接続可能性を評価し「短絡」を発見（multi-hop）
- ✅ 構造的な「洞察」を得てバックトラック（BT）
- ✅ 効率的に探索空間を縮小しながらゴールへ到達
