# 初期人工知能の設計思想

## 1950年代：記号主義の時代

### Logic Theorist (1956) - 最初のAIプログラム
**作者**: Allen Newell, Herbert Simon
**設計思想**:
```
人間の思考 = 記号の操作
数学の証明 = ルールの適用
```
**実装**:
- 公理から定理を導出
- 探索木を使った証明探索
- ヒューリスティックで枝刈り

### General Problem Solver (1959)
**設計思想**: 「手段-目的分析」
```
1. 現状と目標の差分を計算
2. 差分を減らす操作を選択
3. 再帰的に適用
```
**興味深い点**: geDIGの「差分」概念に通じる！

## 1960-70年代：知識表現の探求

### SHRDLU (1970)
**作者**: Terry Winograd
**設計思想**:
```
言語理解 = 世界モデルの操作
積み木の世界で自然言語対話
```
**特徴**:
- 限定世界での完全な理解
- 文脈を考慮した対話

### Expert Systems (1970s-)
**代表**: MYCIN（医療診断）
**設計思想**:
```
IF (症状A AND 症状B) THEN (病気X: 確信度0.8)
```
**問題点**:
- ルールの組み合わせ爆発
- 知識獲得ボトルネック

## 設計思想の変遷

### 第1期：論理的思考の模倣
```
前提：思考 = 論理的推論
方法：記号操作、探索、証明
限界：現実世界の曖昧さに対応できない
```

### 第2期：知識の蓄積
```
前提：知能 = 知識量
方法：エキスパートシステム
限界：知識の硬直性、般化の欠如
```

### 第3期：学習へのシフト
```
前提：知能 = パターン認識
方法：ニューラルネット、統計学習
現在：深層学習の隆盛
```

### 2. フレーム問題
- 何を無視すべきか判断できない
- geDIG: 関連性を距離で判断

### 3. 記号接地問題
- 記号と意味の断絶
- 現代: ベクトル埋め込みで解決

## 現代的な再解釈

初期AIの夢：
```
「人間のように考える機械」
```

geDIGの提案：
```
「人間のように洞察する機械」
```

初期AIは「思考の形式」を模倣しようとした。
geDIGは「思考の本質（洞察）」を捉えようとしている。

初期AIの設計思想：
