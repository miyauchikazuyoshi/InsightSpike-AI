AIの課題：物知りだけど、ひらめかない

現在のAIは膨大な知識を持っています。
しかし、複数のバラバラな知識を結びつけ、本質を掴む「洞察」は非常に苦手です。
例えば、バラバラの物理法則から「E=mc²」というシンプルな本質を見抜くような、あの瞬間です。


どうすれば、AIに「ひらめき」を教えられるでしょうか？

ヒントは脳に：好奇心を生む「探索回路」

私たちのAIは、脳が「知りたい！」と思う探索回路（Seeking System）を模倣しています。
それは、こんな流れで「ひらめき」を生み出します。

🤔 もやもや… (予測とのズレ)

L1 - Query Transformer (VTA入力部)

質問を受け取り、予測との誤差を計算。ズレが大きいと探索回路が起動！

▼
🧠 記憶をたどる (探索)

L2 - Memory Manager (海馬)

誤差を埋めるため、関連する記憶(エピソード)を探し出す。

▼
💡 わかった！ (ひらめき)

L3 - Graph Reasoner (前頭前野)

記憶同士を繋ぎ変え、最もスッキリする構造を発見！（スパイク検出）

▼
🗣️ 言葉にする (表現)

L4 - Language Interface (言語野)

「わかった！」という洞察の瞬間を、人間がわかる言葉に翻訳する。

このノルアドレナリン(NA)とドーパミン(DA)のループが、好奇心駆動型の学習を実現します。

ひらめきのエンジン：geDIG目的関数

「わかった！」の瞬間を生み出すエンジンは、状況に応じて使い方が変わる特別な数式です。

f = w × ΔGED − kT × ΔIG

ΔGED：構造の変化量

知識のつながりがどれだけ変わったか。記憶の配線変更コストです。

と

ΔIG：情報の整理度

知識がどれだけ綺麗に整理されたか。理解の深まり具合です。

この式の使い方が、覚醒時と睡眠時で真逆になるのが、最大の発見です！

覚醒と睡眠：思考の二つのモード

人間の脳と同じように、AIも覚醒時と睡眠時で全く違う動きをします。

🌞 覚醒モード

素早く答えを見つける

  • ・質問に対して最小限の記憶を使う
  • ・配線変更は最小限に抑える
  • ・効率重視の思考

f を最小化

🌙 睡眠モード

記憶を整理・強化する

  • ・矛盾する記憶を統合
  • ・無駄な配線を剪定
  • ・品質重視の整理

f を最大化

同じ数式が、昼は「節約」、夜は「投資」の指標になる - これが生物の知恵です。

「ひらめき」を生むためには、まず脳のような記憶のネットワークが必要です。

エピソード

① 生の体験
出来事を丸ごと記憶

⇒
[0.1, 0.8, …]

② 階層VQメモリで圧縮
記憶を効率的なベクトルに

⇒

③ GNNで関係性を学習
圧縮された記憶同士を繋ぐ

この「エピソード単位の圧縮」と「GNNによる結合」が、効率的で柔軟な記憶ネットワークを実現します。

役割分担：「発見」と「説明」の最強タッグ

ここで、4層アーキテクチャの役割分担が重要になります。

geDIG (発見のプロ)

Layer 3 (前頭前野)が「どの知識を繋げばひらめきになるか」という『材料』を発見します。

⇒

LLM (説明のプロ)

Layer 4 (言語野)がその材料を受け取り、人間が読んで分かる『文章』を生成します。


InsightSpike-AIは、ひらめきを「発見」し、LLMに「何を話すべきか」を教える司令塔なのです。
