AIの課題：物知りだけど、ひらめかない

現在のAIは膨大な知識を持っています。
しかし、複数のバラバラな知識を結びつけ、本質を掴む「洞察」は非常に苦手です。
例えば、バラバラの物理法則から「E=mc²」というシンプルな本質を見抜くような、あの瞬間です。


どうすれば、AIに「ひらめき」を教えられるでしょうか？

ひらめきのエンジン：geDIGフレームワーク

「わかった！」の瞬間（L3）は、2つのモノサシで定量的に評価されます。

① 構造のシンプル化 (ΔGED)

知識のつながりがスッキリ単純になるほど高評価。「情報の断捨離」です。

+

② 情報の整理整頓 (ΔIG)

知識が綺麗にグループ分けされるほど高評価。「概念の結晶化」です。

ℱ = ΔGEDnorm − λ·ΔIGnorm

構造（MDL的な複雑さ）と情報（FEP的な驚き/不確実性）を
単一スカラーにブリッジした差分・オンラインの指標です。

【実装解説】ひらめきの土台：記憶の作り方

「ひらめき」を生むためには、まず脳のような記憶のネットワークが必要です。

エピソード

① 生の体験
出来事を丸ごと記憶

⇒
[0.1, 0.8, …]

② エピソード再構成
文脈に応じたベクトル統合

⇒

③ GNNで関係性を学習
再構成された記憶同士を繋ぐ

この「エピソード再構成」と「GNNによる結合」が、効率的で柔軟な記憶ネットワークを実現します。

ひらめきが駆動する「記憶の進化」

高い「ひらめき度」が検出されると、それをトリガーに、AIは自らの記憶ネットワークを能動的に再編成します。

💡

統合 (Merge)

バラバラの知識が一つになる

💥

分裂 (Split)

一つの知識が文脈に応じて分化する

🕸️

再編成 (Reorganize)

知識体系全体が新しい視点で見直される

これにより、AIは単に記憶するだけでなく、経験から学び、概念を成長させることができます。

【フロー】エピソードのみで進む制御（迷路）

1. 観測

現在位置・壁情報を取得し、エピソードを生成

⟶

2. クエリ比較

直近のエピソード群と比較し、未探索経路があれば候補化

⟶

3. 配線評価

仮エッジを追加して geDIG を計算（ΔGED−λΔIG）

⟶

4a. 進行

geDIG ≤ θwire なら配線を採用。未探索へ一歩前進

⟶

4b. バックトラック

過去エピソードがクエリに近づき、行き止まり傾向なら θbt を下回り戻る

⟶

5. 行き止まり検出

geDIGがグラフ短絡（冗長な近道）を検出 → 反転/回避

geDIGは過去エピソードによる短絡・冗長化を監視し、行き止まりや無駄配線を検出。配線閾値（θwire）とバックトラック閾値（θbt）で分岐。

役割分担：「発見」と「説明」のタッグ

ここで、5層（L0は任意）アーキテクチャの役割分担が重要になります。

geDIG (発見のプロ)

Layer 3 (前頭前野)が「どの知識を繋げばひらめきになるか」という『材料』を発見します。

⇒

LLM (説明のプロ)

Layer 4 (言語野)がその材料を受け取り、人間が読んで分かる『文章』を生成します。


InsightSpike-AIは、ひらめきを「発見」し、LLMに「何を話すべきか」を教える司令塔なのです。

Appendix: 関連仮説（参考）

Transformerの成功は、実はgeDIG理論のマイクロスケール実装と解釈できます。

Transformerの要素geDIG理論での「正体」
トークン (Token)構造(GED) + 情報(IG)を持つ、エネルギーの最小単位。
自己注意 (Self-Attention)エネルギーが最小になる安定経路を探すプロセス。
（softmaxは統計力学のボルツマン分布と完全に同型）
マルチヘッド (Multi-head)異なるスケール（温度）で同時にエネルギー地形を観測する仕組み。
geDIG (普遍理論) → InsightSpike (マクロ実装) → Transformer (マイクロ実装/制約付き)

この統一的視点は、Transformerの限界を超え、より柔軟で強力な次世代AIへの道筋を示しています。
